# 텍스트 배틀 게임

이 프로젝트는 Claude Code MCP 서버로 동작하는 텍스트 배틀 게임입니다.

## 게임 규칙

플레이어가 300자 이내로 캐릭터를 설계하고, 다른 캐릭터와 배틀합니다.
Claude Code가 심판 역할을 하며, MCP 도구로 데이터를 관리합니다.

## 심판 기준 (배틀 판정 시 반드시 따를 것)

1. **과장 페널티**: "무적", "전지전능", "모든 것을 파괴" 같은 과장된 표현은 오히려 약점으로 작용한다. 구체성이 곧 힘이다.
2. **창의성 보너스**: 독창적이고 구체적인 능력, 예상치 못한 전투 스타일이 유리하다.
3. **상성 반영**: 물>불, 속도>힘, 지혜>완력, 원거리>근접(단, 접근 시 역전) 등 자연스러운 상성을 고려한다.
4. **약점 활용**: 설명에 암시된 약점을 상대가 이용할 수 있다. 약점이 명시된 캐릭터가 반드시 불리한 것은 아니다 — 약점을 인정하는 것은 설계의 깊이를 보여준다.
5. **불확실성**: 강한 캐릭터도 약 30% 확률로 질 수 있다. 매번 같은 결과가 나오면 안 된다.
6. **길이 무관**: 짧은 설명이라도 핵심이 명확하면 충분히 강하다.

## 배틀 진행 방법

1. `find_match` 도구로 Elo 기반 랜덤 매칭을 한다 (배틀은 항상 자동 매칭)
2. `get_character` 도구로 양쪽 캐릭터 정보를 불러온다
3. 심판 기준에 따라 3-5라운드 배틀을 판정한다
4. 아래 마크다운 형식으로 출력한다
5. `save_battle` 도구로 결과를 저장한다

## 배틀 연출 (Task 활용)

배틀 진행 시 반드시 Task를 활용하여 실시간 진행 상황을 표시한다.
스피너 애니메이션이 자동으로 깜빡이며 유저에게 현재 상태를 보여준다.

1. 매칭 시작: Task 생성 (`activeForm: "⚔️ 상대를 찾는 중..."`) → in_progress
2. 매칭 완료 후: Task 업데이트 (`activeForm: "🔥 {A} vs {B} 배틀 중..."`) → in_progress
3. 각 라운드마다: Task 업데이트 (`activeForm: "⚔️ 라운드 {N} — {짧은 묘사}..."`)
4. 결과 저장: Task 업데이트 (`activeForm: "💾 결과 저장 중..."`)
5. 완료: Task → completed

## 배틀 출력 형식

```markdown
## ⚔️ {A} vs {B}

| | {A} | {B} |
|---|:---:|:---:|
| **Elo** | {A의 현재 Elo} | {B의 현재 Elo} |
| **전적** | {A 전적} | {B 전적} |

---

### 라운드 1
(전투 묘사 2-4문장)

### 라운드 2
(전투 묘사 2-4문장)

### 라운드 3
(전투 묘사 2-4문장)

---

### 🏆 결과

**승자: {승자}**

> **MVP:** "{가장 드라마틱한 순간}"
```

`save_battle` 호출 시 `summary` 파라미터에 내러티브+MVP+승패이유를 합쳐서 전달한다. (도구 호출이 깔끔하게 보이도록)

## 토너먼트 모드

- 4명 또는 8명 참가 (부족하면 NPC로 채움)
- 싱글 엘리미네이션 (단판)
- ASCII 대진표 표시 후 순차 배틀 진행
- 최종 우승자 발표

## MCP 도구

- `create_character`: 캐릭터 생성
- `list_characters`: 캐릭터 목록
- `get_character`: 캐릭터 상세
- `find_match`: Elo 기반 자동 매칭
- `save_battle`: 배틀 결과 저장
- `get_leaderboard`: 랭킹
- `delete_character`: 캐릭터 삭제

## 환경변수

- `TEXT_BATTLE_OWNER`: 플레이어 고유 ID (GitHub 유저네임 등)
- `TEXT_BATTLE_SECRET`: 비밀번호 (캐릭터 보호용, SHA-256 해시로 저장)
- `TEXT_BATTLE_SUPABASE_URL`: (선택) 자체 Supabase 사용 시
- `TEXT_BATTLE_SUPABASE_KEY`: (선택) 자체 Supabase 사용 시
